# 4.4 实时协作与版本管理

实时协作与版本管理是ScholarForge系统的重要功能，支持多人同时编辑论文并追踪修改历史。本节详细介绍CRDT算法的应用、版本控制机制以及冲突处理策略。

## 4.4.1 CRDT算法

### （1）Yjs CRDT简介

系统采用**Yjs（Y-JavaScript）**库实现协作编辑的冲突-free复制数据类型（CRDT）。

**CRDT核心特性**：

- **强最终一致性**：所有节点最终达到一致状态，无需中心化协调
- **本地优先**：操作先在本地执行，再异步同步到其他节点
- **无锁并发**：无需加锁即可实现并发编辑
- **离线支持**：离线编辑，上线后自动合并

**Yjs架构**：

```
用户操作 → Yjs Document → 操作转换 → 本地状态更新 → 网络同步 → 其他节点
                ↑
         持久化存储（IndexedDB）
```

### （2）数据结构

**Y.Text：协作文本**

用于表示论文内容：

```typescript
// 创建Y.Text实例
const yText = new Y.Text()

// 插入文本
yText.insert(0, '第一章 绪论')

// 删除文本
yText.delete(0, 3)

// 格式化
yText.format(0, 4, { bold: true })
```

**Y.Array：协作数组**

用于表示章节列表：

```typescript
// 创建Y.Array实例
const yChapters = new Y.Array()

// 添加章节
yChapters.push([{
  id: 'ch1',
  title: '绪论',
  content: new Y.Text()
}])

// 移动章节
yChapters.move(0, 2)
```

**Y.Map：协作映射**

用于表示文档元数据：

```typescript
// 创建Y.Map实例
const yMetadata = new Y.Map()

// 设置元数据
yMetadata.set('title', '基于多Agent协同的智能科研论文写作系统项目管理研究')
yMetadata.set('author', '王小明')
yMetadata.set('lastModified', Date.now())
```

### （3）操作转换机制

**操作类型**：

Yjs支持三种基本操作：

| 操作 | 说明 | 示例 |
|------|------|------|
| Insert | 插入内容 | 在位置10插入"研究背景" |
| Delete | 删除内容 | 删除位置5-8的内容 |
| Format | 格式化 | 将位置0-4设为粗体 |

**冲突解决**：

当两个用户同时修改同一位置时，Yjs自动解决冲突：

```typescript
// 用户A在位置0插入"研究"
yText.insert(0, '研究')

// 用户B同时在位置0插入"分析"
yText.insert(0, '分析')

// Yjs自动合并：根据用户ID排序，确定插入顺序
// 结果可能是"研究分析"或"分析研究"，但双方一致
```

**位置映射**：

使用"相对位置"而非绝对位置，确保操作在同步后正确应用：

```typescript
// 创建相对位置标记
const relativePosition = Y.createRelativePositionFromAbsoluteIndex(
  yText,
  10  // 绝对位置
)

// 在其他文档中还原位置
const absoluteIndex = Y.createAbsolutePositionFromRelativePosition(
  relativePosition,
  yDoc
)
```

### （4）持久化存储

**IndexedDB持久化**：

将Yjs文档状态持久化到浏览器本地存储：

```typescript
class YjsPersistence {
  private db: IDBDatabase

  async init() {
    return new Promise((resolve, reject) => {
      const request = indexedDB.open('ScholarForgeDB', 1)

      request.onupgradeneeded = (event) => {
        const db = event.target.result
        db.createObjectStore('documents', { keyPath: 'id' })
      }

      request.onsuccess = (event) => {
        this.db = event.target.result
        resolve()
      }
    })
  }

  async saveDocument(docId: string, yDoc: Y.Doc) {
    const transaction = this.db.transaction(['documents'], 'readwrite')
    const store = transaction.objectStore('documents')

    // 将Yjs文档编码为Uint8Array
    const state = Y.encodeStateAsUpdate(yDoc)

    await store.put({
      id: docId,
      state: state,
      timestamp: Date.now()
    })
  }

  async loadDocument(docId: string): Promise<Y.Doc> {
    const transaction = this.db.transaction(['documents'], 'readonly')
    const store = transaction.objectStore('documents')

    const data = await store.get(docId)
    if (!data) return null

    // 解码Yjs文档
    const yDoc = new Y.Doc()
    Y.applyUpdate(yDoc, data.state)

    return yDoc
  }
}
```

## 4.4.2 版本控制

### （1）版本存储策略

**增量存储**：

只存储版本间的差异（update），而非完整内容：

```typescript
class VersionManager {
  private versions: Map<string, Version> = new Map()

  createVersion(
    documentId: string,
    yDoc: Y.Doc,
    metadata: VersionMetadata
  ): Version {
    // 获取自上一个版本以来的更新
    const update = Y.encodeStateAsUpdate(yDoc)

    const version: Version = {
      id: generateUUID(),
      documentId,
      timestamp: Date.now(),
      update,  // 增量更新数据
      metadata: {
        author: metadata.author,
        message: metadata.message,
        ...metadata
      }
    }

    this.versions.set(version.id, version)
    return version
  }

  // 恢复到指定版本
  async restoreVersion(versionId: string): Promise<Y.Doc> {
    const version = this.versions.get(versionId)
    if (!version) throw new Error('Version not found')

    const yDoc = new Y.Doc()
    Y.applyUpdate(yDoc, version.update)

    return yDoc
  }
}
```

### （3）分支与合并

**分支管理**：

支持论文的多版本并行编辑：

```typescript
class BranchManager {
  private branches: Map<string, Branch> = new Map()

  createBranch(
    parentBranchId: string,
    newBranchName: string,
    yDoc: Y.Doc
  ): Branch {
    const parent = this.branches.get(parentBranchId)

    const branch: Branch = {
      id: generateUUID(),
      name: newBranchName,
      parentId: parentBranchId,
      baseVersion: parent.headVersion,
      headVersion: parent.headVersion,
      yDoc: Y.Doc // 从父分支复制
    }

    this.branches.set(branch.id, branch)
    return branch
  }

  // 合并分支
  async mergeBranch(
    sourceBranchId: string,
    targetBranchId: string
  ): Promise<MergeResult> {
    const source = this.branches.get(sourceBranchId)
    const target = this.branches.get(targetBranchId)

    // 获取两个分支的差异
    const sourceUpdate = Y.encodeStateAsUpdate(source.yDoc)
    const targetUpdate = Y.encodeStateAsUpdate(target.yDoc)

    // 尝试自动合并
    const mergedDoc = new Y.Doc()
    Y.applyUpdate(mergedDoc, sourceUpdate)
    Y.applyUpdate(mergedDoc, targetUpdate)

    // 检查冲突
    const conflicts = this.detectConflicts(source, target)

    if (conflicts.length === 0) {
      // 无冲突，自动合并成功
      target.yDoc = mergedDoc
      target.headVersion = this.createVersion(target.id, mergedDoc)

      return { success: true, conflicts: [] }
    } else {
      // 有冲突，需要手动解决
      return { success: false, conflicts }
    }
  }
}
```

### （4）历史版本管理

**版本树结构**：

```
初始版本 (v1)
    ├── 修订1 (v2)
    │       ├── 修订3 (v4)
    │       └── 分支A (v5)
    └── 修订2 (v3)
            └── 分支B (v6)
```

**版本导航**：

```typescript
class VersionNavigator {
  // 获取版本历史
  getVersionHistory(documentId: string): Version[] {
    return Array.from(this.versions.values())
      .filter(v => v.documentId === documentId)
      .sort((a, b) => a.timestamp - b.timestamp)
  }

  // 比较两个版本
  async compareVersions(
    versionId1: string,
    versionId2: string
  ): Promise<DiffResult> {
    const doc1 = await this.restoreVersion(versionId1)
    const doc2 = await this.restoreVersion(versionId2)

    const content1 = doc1.getText('content').toString()
    const content2 = doc2.getText('content').toString()

    return this.diffCalculator.calculateDiff(content1, content2)
  }

  // 回滚到指定版本
  async rollbackToVersion(versionId: string): Promise<void> {
    const version = this.versions.get(versionId)
    if (!version) throw new Error('Version not found')

    // 创建回滚记录
    const rollbackVersion = this.createVersion(
      version.documentId,
      version.yDoc,
      {
        message: `Rollback to version ${versionId}`,
        type: 'rollback'
      }
    )

    // 应用旧版本状态
    await this.applyVersion(versionId)
  }
}
```

## 4.4.3 冲突处理

### （1）协作冲突类型

**同时编辑冲突**：
两个用户同时编辑同一位置。

**解决方案**：
Yjs自动合并，保留双方修改：

```typescript
// 用户A输入："研究方法"
// 用户B输入："研究背景"
// 结果："研究方法研究背景"（根据用户ID排序）
```

**格式化冲突**：
一个用户修改内容，另一个用户修改格式。

**解决方案**：
内容和格式独立处理：

```typescript
// 用户A：将"引言"改为"第一章 引言"
// 用户B：将"引言"设为粗体
// 结果："第一章 引言"（粗体）
```

**结构冲突**：
一个用户删除段落，另一个用户在该段落内编辑。

**解决方案**：
保留编辑内容，但标记为"悬浮"状态：

```typescript
class StructureConflictResolver {
  resolve(yDoc: Y.Doc, conflictingOps: Operation[]) {
    for (const op of conflictingOps) {
      if (op.type === 'delete' && this.hasConcurrentEdits(op)) {
        // 将被删除内容的编辑提取出来
        const floatingEdits = this.extractEdits(op.target)

        // 创建注释标记
        this.createConflictAnnotation(yDoc, floatingEdits)

        // 通知用户处理
        this.notifyUser({
          type: 'structure_conflict',
          message: '有编辑内容因段落删除而悬浮，请确认是否保留'
        })
      }
    }
  }
}
```

### （3）离线编辑处理

**离线队列**：
缓存离线期间的操作：

```typescript
class OfflineQueue {
  private queue: Operation[] = []
  private isOnline: boolean = true

  enqueue(operation: Operation) {
    if (!this.isOnline) {
      this.queue.push(operation)
    }
  }

  async sync() {
    if (this.queue.length === 0) return

    // 按顺序发送缓存的操作
    for (const op of this.queue) {
      await this.sendOperation(op)
    }

    this.queue = []
  }

  setOnlineStatus(online: boolean) {
    this.isOnline = online
    if (online) {
      this.sync()
    }
  }
}
```

**本节小结**：

本节详细介绍了实时协作与版本管理的实现。首先，在CRDT算法方面，采用Yjs库实现协作编辑的冲突-free复制数据类型，支持Y.Text、Y.Array、Y.Map等数据结构，通过操作转换机制实现无锁并发编辑，并使用IndexedDB进行持久化存储。其次，在版本控制方面，采用增量存储策略保存版本历史，实现了Myers Diff算法进行版本对比，支持分支创建与合并，以及完整的版本导航和回滚功能。最后，在冲突处理方面，实现了同时编辑、格式化和结构冲突的自动或手动解决方案，通过Awareness系统同步光标和选区，以及离线编辑队列处理网络中断场景。这些技术共同构成了系统的实时协作与版本管理能力，为用户提供流畅的多人协作写作体验。
