# 文献推荐系统概念设计文档

## 1. 系统概述

### 1.1 目标
为工程管理领域研究生（特别是浙江大学MEM学生）提供个性化的每日文献推荐服务，基于用户研究方向（AI协同项目管理）、学校背景（浙江大学）和专业要求（MEM毕业标准）进行智能化文献筛选与推送。

### 1.2 核心功能
- 多源学术数据库集成（CNKI、Web of Science、IEEE Xplore）
- 个性化推荐算法（基于用户画像的文献匹配）
- 每日定时推送（1-2篇高相关度文献）
- 文献结构化分析（研究背景、核心观点、方法论、可借鉴部分）
- 写作参考示例生成

| **返回格式** | JSON |

### 2.2 Web of Science Starter API
| 项目 | 详情 |
|------|------|
| **基础URL** | `https://api.clarivate.com/apis/wos-starter/v1` |
| **认证方式** | API Key (X-ApiKey header) |
| **主要接口** | `/documents` (文献检索)<br>`/journals` (期刊信息) |
| **支持字段** | TI(标题)、AU(作者)、PY(出版年)、SO(来源)、TS(主题词)等 |
| **返回格式** | JSON |
| **功能特点** | 英文文献为主，包含被引次数、期刊影响因子等指标 |
| **访问计划** | 免费试用(50次/天)、机构会员(5,000次/天)、集成计划(20,000次/天) |
| **适用场景** | 国际高水平期刊、SCI/SSCI论文、英文研究前沿 |

### 2.3 IEEE Xplore API

## 3. 推荐算法逻辑设计

### 3.1 用户画像建模
```python
user_profile = {
    "major": "MEM",                    # 专业：工程管理硕士
    "university": "Zhejiang University", # 学校：浙江大学
    "research_direction": "AI协同项目管理", # 研究方向
    "graduation_requirements": {       # 毕业要求（来自浙大MEM培养方案）
        "theoretical_foundation": True,
        "practical_application": True,
        "innovation_ability": True
    },
    "preferences": {
        "language": ["中文", "英文"],   # 语言偏好
        "publication_type": ["期刊论文", "会议论文"],
        "recency_weight": 0.7,         # 时效性权重（0-1）
        "authority_weight": 0.8,       # 权威性权重（0-1）
        "relevance_weight": 0.9        # 相关度权重（0-1）
    }
}
```

### 3.2 多维度评分体系
每篇文献的综合推荐分数由以下维度加权计算：

**总分 = Σ(维度分数 × 维度权重)**

| 维度 | 权重 | 计算方式 | 说明 |
|------|------|----------|------|
| **相关度匹配** | 0.35 | 关键词匹配度 × 主题词相似度 | 基于TF-IDF或BERT计算文献与用户研究方向匹配度 |
| **时效性** | 0.25 | 1 - (当前年 - 出版年)/10 | 近3年文献得分高，超过10年得分低 |
| **权威性** | 0.20 | 期刊影响因子 × 0.5 + 被引次数 × 0.5 | 归一化处理，高影响因子和高被引文献得分高 |
| **实践价值** | 0.10 | 方法论可借鉴性 + 案例丰富度 | 评估文献对工程管理实践的指导意义 |
| **学校关联** | 0.05 | 作者机构包含"浙江大学"则得1分，否则0分 | 优先推荐本校师生研究成果 |
| **可读性** | 0.05 | 摘要完整性 + 图表质量 | 评估文献的易读性和结构清晰度 |

### 3.3 个性化推荐流程
1. **查询构建阶段**
   - 根据用户研究方向生成中英文关键词组合
   - 添加专业领域限定词（"工程管理"、"项目管理"）
   - 设置时间范围过滤（默认近5年，用户可调）

2. **多源并行检索**
   ```python
   # 伪代码示例
   queries = {
       "cnki": "AI协同项目管理 AND 工程管理",
       "wos": "TS=(AI collaboration project management) AND WC=(Engineering)",
       "ieee": "metadata=(AI project management) AND publication_year>2021"
   }
   results = parallel_search(queries)
   ```

3. **去重与筛选**
   - 基于DOI、标题相似度去除重复文献
   - 过滤非学术类型（新闻、简报等）
   - 排除用户已阅读/已收藏文献（通过状态库记录）

4. **评分排序**
   - 对剩余文献应用多维度评分体系
   - 综合得分降序排列
   - 选取前10%作为候选推荐池

5. **多样性平衡**
   - 确保中英文文献比例（默认1:1）
   - 混合期刊论文和会议论文
   - 平衡理论研究和实践案例

### 3.4 每日推送策略
- **推送时间**：每日上午9:00（用户可自定义）
- **推送数量**：1-2篇（默认1篇中文+1篇英文）
- **推送内容**：
  1. 文献基本信息（标题、作者、来源、摘要）
  2. 结构化分析（研究背景、核心观点、方法论、图表结构）
  3. 写作参考示例（如何在自己的论文中引用该文献的观点）
  4. 个性化解读（该文献与用户研究方向的关联点）

### 4.3 核心组件说明

**1. 用户画像管理器**
- 存储用户基本信息、研究方向、偏好设置
- 支持动态更新（基于用户反馈调整权重）
- 与浙大MEM培养方案数据库联动

**2. 推荐算法引擎**
- 实现多维度评分算法
- 支持A/B测试和算法优化
- 提供推荐解释（为什么推荐这篇文献）

**3. API适配器层**
- 统一接口封装，隔离不同API的差异
- 错误处理和重试机制
- 缓存策略减少API调用次数

**4. 推送调度器**
- 定时任务管理（cron表达式）
- 推送渠道适配（邮件、微信、App通知）
- 失败重试和监控告警

**5. 状态库管理器**
- 文献去重（基于DOI、标题相似度）
- 用户阅读历史记录
- 推送效果追踪（点击率、收藏率）

### 5.2 数据存储设计

**用户表 (users)**
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE,
    email VARCHAR(100) UNIQUE,
    major VARCHAR(50),           -- 专业
    university VARCHAR(100),     -- 学校
    research_direction TEXT,     -- 研究方向
    preferences JSONB,           -- 偏好设置
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**文献元数据表 (articles)**
```sql
CREATE TABLE articles (
    id SERIAL PRIMARY KEY,
    doi VARCHAR(100) UNIQUE,
    title TEXT NOT NULL,
    authors JSONB,              -- 作者列表
    source VARCHAR(200),        -- 来源期刊/会议
    publish_year INTEGER,
    abstract TEXT,
    keywords JSONB,             -- 关键词列表
    citation_count INTEGER,
    source_db VARCHAR(20),      -- 来源数据库：cnki/wos/ieee
    raw_data JSONB,             -- 原始API响应数据
    indexed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**推荐记录表 (recommendations)**
```sql
CREATE TABLE recommendations (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    article_id INTEGER REFERENCES articles(id),
    recommendation_score DECIMAL(5,4),
    pushed_at TIMESTAMP,
    user_feedback VARCHAR(20),  -- clicked/saved/ignored
    feedback_at TIMESTAMP
);
```

### 5.3 API接口设计

**用户管理接口**
- `POST /api/users/register` - 用户注册
- `POST /api/users/login` - 用户登录
- `PUT /api/users/profile` - 更新用户画像

**推荐系统接口**
- `GET /api/recommendations/daily` - 获取每日推荐
- `POST /api/recommendations/feedback` - 提交反馈
- `GET /api/recommendations/history` - 查看推荐历史

**文献检索接口**
- `GET /api/articles/search` - 多源文献搜索
- `GET /api/articles/{id}` - 获取文献详情
- `GET /api/articles/analysis` - 文献结构化分析

### 5.4 安全与性能考虑

**安全措施**
- API密钥轮换机制（每90天）
- 请求频率限制（防滥用）
- 数据加密传输（TLS 1.3）
- 用户隐私保护（GDPR合规）

**性能优化**
- 多级缓存策略（Redis + CDN）
- 数据库读写分离
- 异步处理耗时操作
- API响应压缩（gzip）
