# ScholarForge 一键启动脚本说明

> 文档日期：2026-03-04

---

## 📦 脚本列表

| 脚本名称 | 功能 | 适用场景 |
|---------|------|---------|
| `start-all.bat` | 一键启动所有服务 | 最常用，推荐 |
| `start-advanced.bat` | 高级启动工具 | 需要选择启动模式 |
| `stop-all.bat` | 停止所有服务 | 关闭服务 |
| `check-status.bat` | 检查服务状态 | 查看服务运行状态 |

---

## 🚀 快速开始

### 方法一：一键启动（推荐）

双击运行 `start-all.bat`，将自动：

1. ✅ 检查环境
2. ✅ 启动AI服务（端口8004）- 已配置阶跃星辰
3. ✅ 启动API网关（端口8000）
4. ✅ 启动前端服务（端口5173）
5. ✅ 自动打开浏览器访问前端

### 方法二：高级模式

双击运行 `start-advanced.bat`，可选择：

- 启动所有服务
- 仅启动后端
- 仅启动前端
- 启动单个微服务
- 查看服务状态
- 查看日志

---

## 📋 各脚本详细说明

### 1. start-all.bat（一键启动）

最简单的启动方式，适合日常使用。

**启动顺序：**
1. AI服务（端口8004）- 等待0秒
2. API网关（端口8000）- 等待2秒
3. 前端服务 - 等待2秒

**日志位置：** `logs/YYYYMMDD_HHMMSS_*.log`

### 2. start-advanced.bat（高级启动）

提供菜单选择，适合开发调试。

**菜单选项：**
```
[1] 启动所有服务（完整模式）
[2] 仅启动后端服务（AI + 网关）
[3] 仅启动前端服务
[4] 启动单个微服务（选择）
[5] 停止所有服务
[6] 检查服务状态
[7] 查看日志
[8] 退出
```

### 3. stop-all.bat（停止服务）

一键停止所有服务。

**停止范围：**
- AI服务（端口8004）
- API网关（端口8000）
- 前端服务（npm/vite）

### 4. check-status.bat（检查状态）

检查各服务是否正常运行。

**检查内容：**
- AI服务健康状态
- API网关健康状态
- 前端服务响应状态

## ⚙️ 环境要求

### 必需

1. **Python 3.10+** 和虚拟环境
   ```bash
   cd backend
   python -m venv venv
   ```

2. **Node.js 18+** 和 npm
   ```bash
   # 安装依赖
   cd frontend
   npm install
   ```

3. **PostgreSQL** 和 **Redis**
   - 确保服务已启动
   - 数据库已创建

### 已配置（在.env文件中）

- ✅ 阶跃星辰API Key
- ✅ 数据库密码
- ✅ JWT密钥
- ✅ MinIO密钥

## 🐛 故障排除

### 问题1：提示"未找到Python虚拟环境"

**原因：** 虚拟环境未创建

**解决：**
```bash
cd backend
python -m venv venv
```

### 问题2：提示"未找到npm"

**原因：** Node.js未安装

**解决：**
访问 https://nodejs.org/ 下载并安装 Node.js 18+

### 问题3：端口被占用

**现象：** 服务启动失败，提示端口占用

**解决：**
1. 运行 `stop-all.bat` 停止所有服务
2. 或手动关闭占用端口的程序
3. 重新运行 `start-all.bat`

### 问题4：AI服务启动失败

**检查：**
1. 查看 `.env` 文件中的 `STEPFUN_API_KEY` 是否配置
2. 检查网络连接（能否访问 https://api.stepfun.com/）
3. 查看日志文件 `logs/ai-*.log`

## 📝 日志说明

日志文件保存在 `logs/` 目录下：

```
logs/
├── ai-20240304_120000.log       # AI服务日志
├── gateway-20240304_120000.log  # 网关日志
└── frontend-20240304_120000.log # 前端日志
```

查看日志：
```bash
# 查看最新日志
type logs\ai-*.log

# 实时查看日志（在终端中）
tail -f logs\ai-*.log
```
