## 4. 知识图谱架构

### 4.1 图谱数据模型

```
┌─────────────────────────────────────────────────────────────────┐
│                    Neo4j 知识图谱模型                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  节点类型 (Nodes):                                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│  │  Concept    │  │  Paper      │  │  Author     │            │
│  │  (概念)      │  │  (论文)      │  │  (作者)      │            │
│  └─────────────┘  └─────────────┘  └─────────────┘            │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│  │  Journal    │  │  Institution│  │  Keyword    │            │
│  │  (期刊)      │  │  (机构)      │  │  (关键词)    │            │
│  └─────────────┘  └─────────────┘  └─────────────┘            │
│                                                                 │
│  关系类型 (Relationships):                                      │
│  ┌───────────────────────────────────────────────────────┐    │
│  │ Paper -[:CITES]-> Paper         (引用关系)             │    │
│  │ Paper -[:HAS_KEYWORD]-> Keyword (关键词关联)           │    │
│  │ Paper -[:PUBLISHED_IN]-> Journal (发表期刊)            │    │
│  │ Author -[:WROTE]-> Paper        (作者关系)             │    │
│  │ Concept -[:RELATED_TO]-> Concept (概念关联)            │    │
│  │ Paper -[:USES_METHOD]-> Concept  (方法使用)            │    │
│  └───────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 知识图谱查询示例

```cypher
// 查找与用户研究方向相关的论文及其引用网络
MATCH (p:Paper)-[:HAS_KEYWORD]->(k:Keyword)
WHERE k.name IN ['AI', '项目管理', '协同']
MATCH (p)-[:CITES*1..3]->(cited:Paper)
RETURN p, cited
ORDER BY cited.citation_count DESC
LIMIT 50

// 发现研究空白：高被引但近期研究较少的方向
MATCH (c:Concept)<-[:HAS_KEYWORD]-(p:Paper)
WHERE p.year >= 2020
WITH c, count(p) as recent_count
MATCH (c)<-[:HAS_KEYWORD]-(old_p:Paper)
WHERE old_p.year < 2020
WITH c, recent_count, count(old_p) as old_count
WHERE recent_count < old_count * 0.3
RETURN c.name as potential_gap, old_count, recent_count
```

## 6. 安全架构

```

传输层安全
├──► TLS 1.3加密
├──► HSTS强制HTTPS
└──► 证书自动续期(Let's Encrypt)

网络层安全
├──► 防火墙规则
├──► 网络策略（K8s NetworkPolicy）
├──► DDoS防护
└──► WAF(Web应用防火墙)

数据层安全
├──► 数据加密（AES-256）
├──► 密钥管理（KMS/Vault）
├──► 数据脱敏
└──► 备份加密
```

### 6.2 身份认证流程

```
用户登录请求
    │
    ▼
API Gateway（路由到用户服务）
    │
    ▼
用户服务（验证凭证）
    ├──► 成功：生成JWT Token + Refresh Token
    └──► 失败：返回错误信息
    │
    ▼
返回Token给客户端
    │
    ▼
后续请求携带Token
    │
    ▼
API Gateway（验证Token有效性）
    │
    ├──► Token有效：路由到目标服务
    └──► Token过期：使用Refresh Token刷新
    │
    ▼
路由到目标服务
```

```

├──► Prometheus（指标收集）

└──► 自定义业务指标
     ├──► 活跃用户数

├──► Alertmanager（告警管理）
├──► 分级告警策略（P0/P1/P2）
└──► 多种通知渠道（邮件/钉钉/短信）
```

### 7.2 关键监控指标

| 指标类型 | 指标名称 | 阈值 | 告警级别 |
|----------|----------|------|----------|
